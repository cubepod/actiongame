# ロストゲート・クロニクル (Lost Gate Chronicles) 開発経緯・修正履歴まとめ

本作において実施された、バグ修正、機能追加、バランス調整、およびモバイル対応等の一連の開発経緯と修正内容をここにまとめます。

---

## 1. 不具合・バグの修正

| 修正内容 | 発生原因と対策 |
| :--- | :--- |
| **デバッグ用ダイアログ表示による進行不能バグの解消** | プレイヤーがやられた（残機減少）の際、デバッグ用ダイアログが表示されてゲームが再開できなくなる問題を修正し、スムーズにリスポーンするようにしました。 |
| **リトライ時にゲーム速度が倍増するバグの修正** | ゲームオーバーからのリトライ時に `requestAnimationFrame` が二重起動していたため、ループの単一起動を保証するフラグ `isLoopRunning` および `startLoop()` ヘルパーを実装して完全に解消。 |
| **キー入力なしでの「勝手移動」バグの修正** | ウィンドウのフォーカス喪失時（`blur`）、ステージの切り替え時（`loadStage`）、一時停止時（`togglePause`）に、すべてのキー入力状態 (`keys`) と関連フラグを完全にクリアする処理を追加しました。 |
| **ジャンプ入力の遅延（レイテンシ）解消** | キー押下（`keydown`）および画面タッチ（`touchstart`）の入力イベント内で直接 `player.vy = -9.2` を適用する `triggerJump()` を実装し、フレーム更新のズレ（最大16.6ms）による遅延感をゼロ化しました。 |
| **長押しによる連続ジャンプ（オートリピート）の防止** | 着地時にジャンプキーを押しっぱなしにしていると連続でジャンプしてしまうのを防ぐため、キーリリース監視フラグ `spaceDampened` を導入し、一度キーを離して再押下した時のみジャンプが発動する仕様に修正しました（水中の上昇ホールド操作は維持）。 |
| **音声システム非対応環境でのクラッシュ対策** | 自動テスト環境（Node.jsなど）で音声コンテキスト（Web Audio API）が生成されず `audioCtx` が `null` の場合にクラッシュするのを防ぐため、`playSound()` に `if (!audioCtx) return;` の安全チェックを追加しました。 |

## 2. 新機能の追加

### ① 盾（シールド）アイテム
* **仕様**: 敵の弾を1回だけ防ぐことができる盾を実装しました。
* **出現方法**: 初期装備ではなく、隠し部屋（宝箱）の中に低確率でランダムに出現します。
* **復活防止仕様**: 隠し部屋でアイテム（残機UPや盾）を獲得後、同じステージでプレイヤーがやられても隠し部屋が再出現しない入場制限（1回限り）を適用しました。

### ② スマホ・モバイル端末対応 & スケーリング
* **マルチデバイス対応**: PCからのアクセス時はキーボード、タッチデバイスからのアクセス時は画面上の透過型操作ボタン（左右移動、ツタ登降、JUMP、THROW）でプレイ可能。
* **アスペクト比固定スケーリング**: 画面が縮小した際、アスペクト比 4:3 を維持したまま自動的に画面中央で比率を維持して縮小（`transform: scale()`）する CSS とリサイズ関数を実装。プレイヤーの配置が右端に偏る不具合も中央揃えに修正。

### ③ 水中ステージのギミック（天井ハザードと逆さの敵）
* **天井ハザード**: 水面を泳ぐだけで簡単に敵を避けられるのを防ぐため、天井（y = 60px）に下向きの「天井トゲ」を配置。
* **天井プラットフォーム (柱)**: 天井から下向きに垂れ下がる柱を追加。下端には砂トリムと**下向きに揺れる海藻（苔）**を描画。
* **逆さの敵**: 天井や柱の底面にぶら下がり、逆さの状態で往復歩行したり弾を撃つ敵（スライム、魔法使い、火吹き）を追加。
  * **反転描画**: `scale(1, -1)` を用いて画像パーツを自動上下反転。
  * **踏みつけ無効**: トゲなどの危険部位が下を向いているため、踏みつけによる撃破を無効化（接触時にダメージを受ける）。

### ④ 投擲可能な武器（斧、投槍、ナイフ）
* **武器の特性**:
  * **ナイフ (Knife 🔪)**: ダメージ `1`。重力の影響が極めて低く、水平に超高速で直進。
  * **斧 (Axe 🪓)**: ダメージ `2`。高い放物線を描いてくるくる回転しながら飛ぶ（障害物越しの攻撃に有利）。
  * **投槍 (Javelin 🔱)**: ダメージ `3`。重く直線的な軌道で高速に飛び、高い威力を誇る。
* **出現・所持制限**: 
  * ステージ開始時に60%の確率でスタート地点にランダムで出現（出現しない場合もあり）。
  * プレイヤーがやられた場合、装備していた武器は没収され「武器なし」状態に戻るデスペナルティ仕様。
* **操作性**: PCは `Z` / `X` キー、モバイルは画面上の赤い `THROW` ボタンで投擲。
* **衝突判定**: 壁接触時に火花（破片）を散らして消滅し、敵接触時はダメージを与え、撃破時はポーションドロップ判定。

## 3. バランス調整

### ① 最終ボスの見た目調整
* **課題**: 最終ステージのボス頭部に「大きな角」と「王冠のトゲ」が描画されており、プレイヤーが「頭部にトゲがあり、踏むとダメージを受ける（踏めない）」と誤認する恐れがありました。
* **対策**: ボスの角およびトゲの描画コードを削除し、滑らかで丸みを帯びた頭部形状に変更することで、踏みつけてダメージを与えられるボスであることが直感的にわかるようにしました。

### ② 敵の体力（HP）バリエーション化と無敵時間の実装
* **敵ごとのHP**:
  * 緑スライム: `1 HP`
  * トゲスライム / オレンジジャンパー / 紫魔法使い: `2 HP`
  * 赤火吹き: `3 HP`
  * これにより、3種類の武器（1ダメ〜3ダメ）を使い分ける攻略性が向上しました。
* **踏みつけアクションの変更**:
  * 踏みつけ攻撃のダメージを「1ダメージ」に変更（HPが2以上の敵は複数回踏むか武器の追撃が必要）。
* **被弾点滅と接触ダメージ無効**:
  * ダメージを受けた敵は `hurtTimer` (0.5秒間) が作動して半透明で点滅します。
  * **点滅中はその敵からの接触ダメージ判定（体当たりやトゲ）が完全に無効化**されるため、踏みつけた直後に上昇・離脱するプレイヤーが安全に避難できるようになりました。

### ③ 魔法使いのホーミング弾（魔法弾）の弱体化
* **射程（寿命）の短縮**: 生存時間を180フレーム（約3秒）から **100フレーム（約1.6秒）** に短縮し、一定距離を過ぎると自然消滅するようにしました。
* **追尾範囲の制限**: プレイヤーとの直線距離が **280px未満** の場合のみ追尾角度補正が働き、280px以上の遠距離では直進するよう制限を設け、無限追尾による理不尽な被弾を防止しました。

## 4. 自動テストによる検証履歴

本変更が既存のゲームシステムに影響を与えていないことを保証するため、Node.jsベースの自動検証テスト（モック環境）を整備・実行し、すべてパスすることを確認しました。

* **`test_bugs.js`**: リトライ時の多重ループ防止、ウィンドウフォーカス喪失時の入力リセット、ステージロード時の入力リセット、ワンショットジャンプの動作検証（クリア）。
* **`test_underwater.js`**: 水中ステージにおける天井プラットフォーム、天井トゲ、逆さ敵のランダム生成が正しく実行されるかの検証（クリア）。
* **`test_weapons.js`**: 武器アイテムの出現、装備、斧/槍/ナイフの投擲物理（初期速度・重力）、連投クールダウン制御、死亡リスポーン時の武器消失の検証（クリア）。
* **`test_enemy_hp.js`**: 敵ごとのHP初期設定、踏みつけによるHP減少、被弾点滅中の接触ダメージ無効化、HPゼロ時の撃破判定の検証（クリア）。
* **`test_homing.js`**: 魔法使いの魔法弾の寿命短縮、280px外での追尾機能オフ、280px内での追尾機能オンの検証（クリア）。
